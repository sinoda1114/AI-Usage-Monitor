# AI Usage Monitor（Chrome 拡張）

## 画面イメージ

Chrome の**ツールバーにあるこの拡張のアイコン**を押すと、小さなウィンドウ（ポップアップ）が開きます。そこで Claude・Codex・Cursor（オプションで Devin）の使いみちを、バーや％、リセットの日時などひと目で見られます。

![ポップアップのスクリーンショット。ヘッダーに最終更新と稼働状態、更新・停止・設定ボタン。Claude・Codex・Cursor のカードにメトリクスとリセット時刻が表示されている。](docs/popup-screenshot.png)

## 概要

Cursor・Codex・Claude の公式サイトにある**使用量のページ**に出ている数字を読み取り、**ツールバーのポップアップ**にまとめて表示します（オプションで **Devin** の使用量ページも）。Chrome 拡張、Manifest V3。

ポップアップと設定画面の UI 言語は **English / 日本語 / 简体中文 / 한국어 / Español** から選べます（各サービスの使用量ページ上のラベルは、ページ側の表示言語のままです）。

読み取った内容は**このブラウザの中にだけ**保存します。外部のサーバーへ送ったり、別の端末と同期したりはしません。

[プライバシーポリシー（英語・日本語）](PRIVACY.md) — Chrome Web Store 申請時の URL 用にも使えます。  
**サポート:** 不具合・要望・質問は [GitHub Issues](https://github.com/sinoda1114/AI-Usage-Monitor/issues) まで。

## インストール

まず配布用 ZIP を入手します。

- **Chrome Web Store 申請用 ZIP（`manifest.json` の `version` と一致するファイル名）:** リポジトリの `releases/ai-usage-monitor-store-v0.5.4.zip` を使うか、ルートで `python scripts/package-store-zip.py` を実行して同じものを生成してください（`usage-collector.js`・`i18n.js`・`_locales`・アイコン等を含む申請用の一式です）。
- **手早く直リンクする場合（`main` に ZIP が含まれているとき）:**  
  `https://github.com/sinoda1114/AI-Usage-Monitor/raw/main/releases/ai-usage-monitor-store-v0.5.4.zip`
- **別バージョンや Release 資産を探す場合:** [GitHub の Releases 一覧](https://github.com/sinoda1114/AI-Usage-Monitor/releases)

1. 上のリンクから **ZIP をダウンロードして展開する**（中に `manifest.json` などが並んだフォルダができます）。
2. Chrome のアドレスバーに `chrome://extensions` と入力して開く。
3. 右上の **「デベロッパーモード」** をオンにする。
4. **「パッケージ化されていない拡張機能を読み込む」** を押し、さきほど展開したフォルダ（`manifest.json` が入っている階層）を選ぶ。

## 使い方

1. 拡張を入れると、各サービスの**使用量用のタブ**を開いたうえで（まだ無ければ開き、あれば再読み込みして）数字を取りに行きます。**タブは取得後も閉じません**（開きっぱなしです）。ポップアップで**停止**にしているあいだは巡回しません。**再開**でまた始まります。
2. **ツールバーのアイコン**を押すと、いま保存されている内容がポップアップに出ます。
3. ポップアップの **「設定」** からオプションを開けます。下の画面のとおりです。
   - **表示言語** … English / 日本語 / 简体中文 / 한국어 / Español、またはブラウザに合わせる。
   - **Devin** をポップアップに出したいときは **Devin を表示する** にチェックし、**組織スラッグ**（使用量ページの URL の `/org/` の直後の名前。例: `my-org`）を入力して **保存** してください。既定では Cursor・Codex・Claude のみです。
   - **表示／非表示** … 各行の **「表示する」** のチェックで、ポップアップに出すかどうかを切り替えます。
   - **見分け** … Cursor / Codex / Claude / Devin は、ポップアップと同様の **アイコンと左の色線** で区別できます。
   - **並び替え** … 行全体を **ドラッグ** して並べ替えます（上にあるほどポップアップでも上に表示されます）。
   - **自動更新の間隔** … 1〜120 分を**数字で入力**するか、プリセットから選べます。
   - 変えたあとは **「保存」** を押してください。

![設定画面。Cursor・Codex・Claude に「表示する」チェックと並び順の数字、保存ボタンがある。](docs/options-screenshot.png)

## 注意（必読）

**数字が取れるのは、各サービスの公式の「使用量」ページが、ログインした状態でちゃんと表示されているときだけです。**

- 拡張が裏でタブを開いても、**ログインしていない・時間が切れている・ブロックされている**などでページが空なら、数字は入りません。
- うまく出ないときは、**自分でそのサービスの使用量ページを開いて、画面に数字が出ているか**確かめてください。
- 各サービスが画面の見た目や文言を変えると、この拡張が追いつかず表示されなくなることがあります。
- 拡張を更新したあとは、**使用量タブを F5 で再読み込み**すると、古い収集スクリプトが残りにくくなります。

## 処理の流れ（ざっくり）

下の図の「ポップアップの画面」は、さきほどのスクリーンショットのような**拡張のウィンドウ**のことです。

```mermaid
flowchart TD
  subgraph trigger["きっかけ"]
    A1[拡張のインストール / ブラウザ起動]
    A2[設定した間隔のアラーム]
    A3[ポップアップの「更新」ボタン]
  end

  A1 --> B
  A2 --> B
  A3 --> B

  B{更新停止中?}
  B -->|はい| Z[巡回しない]
  B -->|いいえ| C[background: 各サービスの Usage URL をタブで開く or 再読込]

  C --> D{そのタブで Usage の内容が表示された?}
  D -->|いいえ| E[当該プロバイダー分は未取得のまま]
  D -->|はい| F[usage-collector: 画面からメトリクスを抽出]

  F --> G[background へスナップショット送信]
  G --> H[(このブラウザ内に保存)]

  subgraph popup["ポップアップの画面（スクリーンショットの UI）"]
    direction TB
    P0[ツールバーのアイコンで開く]
    P0 --> P1[見出し AI Usage Monitor・最終更新・稼働中/停止中]
    P1 --> P2[操作: 更新・停止/再開・設定]
    P2 --> P3[プロバイダー別カード Cursor / Codex / Claude / Devin]
    P3 --> P4[各カード内: メトリクス名・リセット時刻・%・プログレスバー・クレジット等]
    P4 --> P5[保存したデータをポップアップに表示]
  end

  H --> P5
  E --> P5
  P2 -.->|「更新」| A3
```

**クラウドに上げたり、別の端末と同期したりはしません。** このブラウザに保存した内容を、ポップアップが見せているだけです。取れていないサービスは、カードが空か、前に取れた数字のままになることがあります。
